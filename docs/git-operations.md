# Git 運用

## この文書の位置づけ

この文書は、リポジトリ上で確認できる事実と、現時点の文書整備前提を最小限でまとめたものです。Git の詳細運用手順はコードから直接は読めないため、断定は避けます。

## 確認できる事実

- このリポジトリは Git 管理されています。
- `main` ブランチが存在します。
- 作業用のトピックブランチが複数存在します。

## 基準点固定

- 今回のドキュメント整備では、現行 `main` を良好な基準点として扱います。
- ドキュメント内容は、この基準点のコードと矛盾しないことを優先します。

## ブランチ作成

- トピックブランチ運用が行われていること自体は、ローカルブランチ一覧から確認できます。
- ただし、ブランチ命名規則や必須手順はコードからは確定できません。

## main へのマージ

- `main` が統合先として存在することは確認できます。
- ただし、レビュー手順、マージ方式、必須チェックの運用はコードからは確定できません。

## 未コミット変更を混ぜないルール

- このルールはコードで強制されるものではありません。
- ただし、基準点との差分を明確に保つため、ドキュメント更新時も対象外変更を混在させない運用が妥当です。

## 不明点

- ブランチ作成の正式手順
- main マージ時の正式手順
- レビュー承認条件
- CI 通過条件

これらはリポジトリ運用ルールとして別途定義が必要です。
