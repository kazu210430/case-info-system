# TaskPane 実機テスト観点チェックリスト

## 1. 前提

- テストの目的
  - TaskPane action 実行フローのリファクタリング後も、現行挙動からの差分が出ていないことを実機で確認する。
- 対象バージョン
  - 実施時点の対象ブランチ名、HEAD コミット、Add-in 配備日時を記録する。
  - 記録欄
    - 対象ブランチ:
    - 対象コミット:
    - Add-in 配備日時:
    - 実施日:
    - 実施者:
- 実施タイミング
  - TaskPane 関連のリファクタリング後
  - `TaskPaneActionDispatcher`、`DocumentCommandService`、`AccountingSetCreateService`、TaskPane refresh / 表示制御まわりを変更した後
  - Debug Add-in を再配備した後
- テスト対象の前提データ
  - `doc` ボタンが 1 件以上表示される既存 CASE を使用する。
  - `会計書類セット` ボタンが表示される同一 CASE を使用する。
  - `案件一覧登録` を実行しても問題ない検証用 CASE を使用する。

## 2. テスト観点

### ■ doc action

- `DOC-01` 文書ボタン押下
  - 表示中 CASE の任意の文書ボタンを 1 回押したとき、処理開始が 1 回だけ発生すること。
- `DOC-02` 文書名入力
  - 文書名入力ダイアログが 1 回だけ表示されること。
  - キャンセル時に Word 起動、TaskPane の不自然な再描画、他 Workbook へのフォーカス移動が発生しないこと。
- `DOC-03` Word 起動
  - 確定後に Word が起動し、生成文書が 1 件だけ表示されること。
- `DOC-04` 表示順序
  - 文書生成直後に CASE 側 TaskPane が即時再描画されず、Word 前面表示が維持されること。
- `DOC-05` フォーカス状態
  - Word を閉じる、または Excel に戻した後も、元の CASE の TaskPane が消失、白表示、二重表示にならないこと。

### ■ accounting action

- `ACC-01` 会計ブック起動
  - `会計書類セット` ボタン押下後、会計ブック作成処理が 1 回だけ開始されること。
- `ACC-02` 前面表示
  - 会計ブックが可視化され、初回表示時に前面へ出ること。
- `ACC-03` CASE との関係
  - 元の CASE が閉じられずに残ること。
  - 会計ブック表示直後に CASE 側 TaskPane が不自然に再描画されないこと。
  - CASE に戻った後も TaskPane が消失、白表示、二重表示にならないこと。

### ■ caselist（凍結領域だが観測対象）

- `CL-01` caselist 表示
  - `案件一覧登録` 押下後、登録完了メッセージが 1 回だけ表示されること。
  - Kernel 側の案件一覧シート表示へ遷移すること。
- `CL-02` 選択変更
  - 案件一覧表示後に別行を選択しても、Excel エラー、二重再描画、白表示が発生しないこと。
- `CL-03` 他 action との干渉がないこと
  - 案件一覧登録後に CASE を再度開いたとき、`doc` / `accounting` / `案件一覧登録` の各ボタン表示が欠落しないこと。
  - CASE を開き直した後も他 action が通常どおり実行できること。

### ■ post-action refresh

- `REF-01` action 実行後の TaskPane 更新
  - `doc` 実行後は Word 前面表示を優先し、CASE 側 TaskPane がその場で強制再描画されないこと。
  - `accounting` 実行後は会計ブック前面表示を優先し、CASE 側 TaskPane がその場で強制再描画されないこと。
  - `caselist` 実行後は CASE 側 pane 署名が更新され、次回表示時に欠落なく再構成されること。
- `REF-02` データ反映
  - CASE に戻ったとき、表示中の TaskPane がその CASE のボタン群を保持していること。
  - 他 CASE のボタン、空 pane、欠落ボタンが表示されないこと。
- `REF-03` 二重実行や欠落がないこと
  - 1 回の押下でメッセージ、Word、会計ブック、一覧遷移が重複しないこと。
  - 実行結果が未表示のまま終わらないこと。

### ■ render / show 順序

- `ORD-01` 表示順序が崩れていないこと
  - CASE を開いた直後、TaskPane が安定表示された後に action を実行できること。
  - CASE 切替、CASE への戻り、会計ブックへの切替後も、別 Workbook 用 pane が混在しないこと。
- `ORD-02` チラつき・白画面の有無
  - CASE 表示時、action 実行時、CASE 復帰時に白 Excel 相当の白表示が出ないこと。
  - TaskPane が点滅、閉じ直し、二重生成のような見え方にならないこと。

## 3. 実施手順

1. Excel を起動する。
2. Add-in が読み込まれていることを確認する。
3. 検証用 CASE を開き、TaskPane が安定表示されるまで待つ。
4. `doc` 観点を確認する。
   - 任意の文書ボタンを 1 回押す。
   - 文書名入力ダイアログをキャンセルし、`DOC-01` と `DOC-02` を確認する。
   - 同じ文書ボタンを再度 1 回押し、今度は確定して `DOC-03` から `DOC-05` を確認する。
5. CASE に戻り、`会計書類セット` ボタンを 1 回押して `ACC-01` から `ACC-03` を確認する。
6. CASE に戻り、`案件一覧登録` ボタンを 1 回押して `CL-01` から `CL-03` を確認する。
7. 案件一覧表示後、元の CASE を再度開く、または同じ CASE を開き直し、`REF-01` から `REF-03` と `ORD-01` から `ORD-02` を確認する。

## 4. 判定基準

- `doc action`
  - OK: ダイアログ、Word 起動、前面表示、CASE 復帰後の TaskPane 状態が各 1 回で期待どおりに観測できる。
  - NG: ダイアログ未表示、二重表示、Word 未起動、Word 以外が前面化、CASE 復帰後の pane 欠落 / 白表示 / 二重表示がある。
- `accounting action`
  - OK: 会計ブックが 1 回だけ生成され、可視化され、前面表示され、CASE 復帰後も pane が壊れない。
  - NG: 会計ブック未生成、二重生成、前面化失敗、CASE 側 pane の欠落 / 白表示 / 二重表示がある。
- `caselist`
  - OK: 完了メッセージが 1 回だけ表示され、案件一覧へ遷移し、再度 CASE を開いても他 action が壊れない。
  - NG: メッセージ未表示 / 二重表示、一覧遷移失敗、一覧表示後の選択変更で異常、CASE 再表示後のボタン欠落がある。
- `post-action refresh`
  - OK: action ごとの foreground 優先挙動が保たれ、戻った時点で正しい TaskPane が 1 つだけ表示される。
  - NG: action 実行直後の過剰 refresh、戻り後の未 refresh、他 CASE の内容混入、二重実行がある。
- `render / show 順序`
  - OK: CASE 表示、action 実行、CASE 復帰のいずれでも白表示、点滅、二重生成がない。
  - NG: 白 Excel 相当、TaskPane の点滅、別 Workbook 用 pane 混在、フォーカス不整合がある。

## 5. 注意事項

- Excel 再起動の必要性
  - 1 回目の観測結果が不安定だった場合は、Excel と Word をいったん終了してから再実施する。
- 既存 CASE の影響
  - `案件一覧登録` は CASE の状態を進めるため、繰り返し確認する場合は検証用コピーを使う。
  - 生成済み文書や生成済み会計ブックが残っていると判定を誤るため、毎回同じ初期状態から始める。
- 環境依存の可能性
  - Word / Excel の前面表示はマルチモニタ、最小化状態、既存 Word プロセスの有無に影響される場合がある。
  - 環境差が疑われる場合も、白表示、二重表示、未表示、別 Workbook 混在の有無は必ず記録する。
- 本チェックリストの対象外
  - 帳票内容そのものの正しさ
  - 会計シート内セル値の業務妥当性
  - 雛形登録・更新フローの正しさ
